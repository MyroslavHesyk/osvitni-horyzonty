# Освітні горизонти 2026 — сайт соцмережевого челенджу

Статичний сайт для GitHub Pages + опціональна база Firebase Firestore.

## Що вже є

- Головна сторінка з описом активності, темами днів і формою додавання поста.
- Адмін-сторінка з підрахунком активності, денними розіграшами, загальним розіграшем і CSV-експортом.
- Сторінка-заготовка для розкладу.
- Локальний демо-режим: якщо Firebase не налаштовано, заявки зберігаються тільки у браузері.

## Як запустити локально

Відкрийте `index.html` у браузері або запустіть локальний сервер:

```bash
python3 -m http.server 8000
```

Після цього відкрийте:

```text
http://localhost:8000
```

## Як опублікувати на GitHub Pages

1. Створіть репозиторій на GitHub.
2. Завантажте всі файли сайту в репозиторій.
3. У Settings → Pages виберіть джерело публікації: `Deploy from a branch`, гілка `main`, папка `/root`.
4. Збережіть налаштування та дочекайтесь публікації.

## Як підключити Firebase

1. Створіть Firebase project.
2. Додайте Web App.
3. Скопіюйте Firebase config у `js/config.js`.
4. Увімкніть Firestore Database.
5. Увімкніть Authentication → Email/Password.
6. Створіть користувача-адміністратора.
7. Вставте правила з `firestore.rules` у Firestore Rules та опублікуйте їх.
8. Після цього форма на сайті буде зберігати заявки у спільну базу, а `admin.html` відкриватиме їх після входу.

## Важливо

Без Firebase сайт працює тільки як демо: заявки зберігаються в `localStorage` одного браузера і не будуть видимі іншим користувачам.
